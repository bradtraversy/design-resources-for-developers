## High‑Level Architecture Overview

- **Framework**: Next.js 16 using the App Router. The codebase follows the new `app/` directory layout rather than the legacy `pages/` approach. See `node_modules/next/dist/docs/` for the official Next.js breaking‑change guide.
- **Language**: TypeScript throughout the repo.
- **Database**: MongoDB accessed via Prisma. The Prisma schema lives in `prisma/schema.prisma`. Generated client is re‑exported from `lib/db.ts`.
- **Styling**: Tailwind 4 with utility‑first CSS.
- **UI Components**: Radix UI primitives, Lucide icons, and a custom component library under `components/`.
- **State Management**: Mostly React hooks; global state (favorites) is persisted in `localStorage` via the `use-favorites` hook.
- **Server‑Side Logic**: Server actions in `app/actions.ts` perform data mutations using the Prisma client.
- **Admin Section**: Protected routes under `/admin/*` provide bulk import, resource submission review, and analytics dashboards.
- **Analytics**: Click tracking updates a `clickCount` field in the DB; the UI can sort by popularity using this metric.
- **Feature Flags**: Dark mode follows the system preference using CSS media queries.

### Key Directories
```
app/                # Next.js App Router pages and server actions

lib/                # Data layer, utilities, types, and custom hooks
prisma/             # Prisma schema and seed script

```
